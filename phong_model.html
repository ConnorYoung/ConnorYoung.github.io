Phong Model Illumination

One of the projects in Concordia’s computer graphics class was to read data from a file and render shapes on the screen using ray casting and the Phong illumination model. To display the images I used OpenGL and the CImg library to help process the images.

Phong Illumination Model

The Phong Illumination Model doesn’t exactly simulate lighting in real life, but it is efficient and looks close enough that it is a very popular model used to render pixels. In order to get the resulting color of the pixel, three colors are added together: the ambient, diffuse and specular color.

  • Diffuse color: the defining color of the object under plain white light.
  • Specular color: The highlights of an object, shiny surfaces have a high specular value.
  • Ambient color: lighting of the scene independent of any light sources.

In this project the diffuse, specular and ambient color of the shape was given in the text file I parsed. However, the color of the pixel to be displayed is also changed by its environment. If a sphere has an object above it then the pixels of the sphere will be a shade darker because they’re in shadow. To compute the color I used ray casting.

Ray Casting

The color of an object relies on light, without light the object wouldn’t be visible. Normally a light would cast out a massive amount of rays lighting any object they intersect with. However, this is really impractical for an application because the rays would miss a lot of objects hitting nothing and wasting computation time. Instead rays are cast out from the camera that we view the scene with. Once a ray intersects with a point we calculate the normal at that point, shoot a shadow ray to see if a shadow is cast onto the object and if it’s unobstructed then we apply the Phong model to it and get a color for the pixel.

The tricky part was finding out whether or not our ray intersected with an object at a given point because the formula for intersection varies between spheres, planes and triangles. For example when rays are cast on a sphere we can use the sphere’s center and radius to find out if it intersects. However, intersection with a triangle is more work because the normal at the vertices can be calculated, but in the middle you need to change the coordinate scheme to barycentric coordinates for linear interpolation between the points.
