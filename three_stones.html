Three Stones

One of the data communications projects at Dawson was to replicate a multiplayer version of the board game Three Stones using networking in Java. We worked on the project in a group of three and used Java FX to create the interface and handle the events.

Networking

The first step is connecting the two players together. At the beginning you need a host to act as the server of the game by launching the server application. The server will automatically host the game using localhost. The client then connects to the game using the server’s IP address.

Once the two players are connected to the game they communicate to each other by exchanging an array of bytes. The first byte is reserved for the instruction, such as a move command or a notification that the game is over. The following bytes were used in our move command to send the row and column of the move.

Once all a player’s stones have been placed a game over instruction will be sent, the winner will be calculated and the winner will display on the screen. The players can then reconnect for a rematch.
